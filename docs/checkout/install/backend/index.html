部署 支付系统后端

注意

支付系统后端至少需要 Java 17 运行。

首先到 Github (opens new window) 下载后端（backend）。像对待 Minecraft 服务端一样对待它，使用以下命令启动后端，并保持其一直在运行。

在插件正式发布之前，请到 Github Actions (opens new window) 下载 SweetCheckout.zip，或自行编译。

java -Xmx1G -jar backend-x.x.x.jar
1

启动后端之后，会生成一个配置文件 config.json，你需要在其中配置各种支付方式。以下是各字段对应的支付服务，以及配置教程。

# 官方 微信 Native

"wechat_native"
1

微信的官方支付接口，点这里 (opens new window)前往申请地址。

  • 微信 Native支付配置

# 官方 支付宝 订单码支付

"alipay_face2face"
1

支付宝的官方支付接口，点这里 (opens new window)前往申请地址。

  • 支付宝 订单码支付配置

# 第三方 Hook

"hook": {
    "wechat"
    "alipay"
}
1
2
3
4

用一些奇奇怪怪的方法变相实现支付接口的效果，不保证其可靠性。

  • 支付宝 Hook配置
  • 微信 Hook配置

# 完成

后端配置完毕后，就可以开始安装并配置插件了。
