在菜单配置的 `items` 中可以添加菜单图标，图标的基本格式如下
```yaml
# 菜单图标列表
items:
  图标1:
    material: WHITE_STAINED_GLASS_PANE
  图标2:
    material: STONE
```

## 基础变量

可以在菜单标题、图标文字中使用的基础变量
+ `%page%` - 当前页码
+ `%max_page%` - 最大页码

## slot/slots

将这个菜单图标放到[界面布局模板](/docs/inventory/menus#inventory)的某个字符所在的位置。
```yaml
    slot: 框
    slots: [ '框' ]
```

## material/data

(可选，默认 `PAPER`) 设置图标的物品类型，可以添加 data 值以兼容 1.12.2 及以下的子 ID 格式。
```yaml
    # 1.13+
    material: PAPER
    # 1.12.2 及以下
    material: WOOL:10
    # 或者这样
    material: WOOL
    data: 10
```

自 `1.0.1` 起，可以在 `material` 使用以下物品格式：  
（基本格式为 `类型:参数`，由 `<>` 包裹的为参数，不需要输入尖括号）
| 格式 | 描述 |
| --- | --- |
| `head:<头颅Base64值>`         | 自定义玩家头颅（可从 [Minecraft Heads](https://minecraft-heads.com/) 等网站中获得 Base64 值） |
| `itemsadder:<命名空间>:<ID>`  | [ItemsAdder](https://itemsadder.devs.beer/) 物品 |
| `craftengine:<命名空间与ID>`  | [CraftEngine](https://ce.gtemc.cn/zh-Hans/) 物品，可选使用 `<ID>` 或 `<命名空间>:<ID>` 两种格式 |
| `mmoitems:<类型>:<ID>`        | [MMOItems](https://gitlab.com/phoenix-dvpmt/mmoitems/-/wikis/home) 物品，展示模式，显示物品属性数值范围 |
| `mmoitems-random:<类型>:<ID>` | [MMOItems](https://gitlab.com/phoenix-dvpmt/mmoitems/-/wikis/home) 物品，随机属性模式，跟给予玩家物品显示的一样 |
| `mythic:<ID>`                | [MythicMobs](https://git.mythiccraft.io/mythiccraft/MythicMobs/-/wikis/home) 物品 |
| `neigeitems:<ID>;<数据值>`    | [NeigeItems](https://github.com/ankhorg/NeigeItems-Kotlin) 物品，分号与数据值是可选的 |
| `nexo:<ID>`                  | [Nexo](https://docs.nexomc.com/) 物品 |

## amount

(可选，默认 `1`) 设置图标的物品显示数量。
```yaml
    amount: 1
```

## display

(可选，默认 `""`) 设置图标的物品显示名称。
```yaml
    display: ''
```

## lore

(可选，默认为空) 设置图标的物品描述。
```yaml
    lore:
      - '第一行'
      - '第二行'
```

## reset-lore

> `1.0.1` 新增

(可选，默认为`false`) 设置是否需要重设物品描述，该选项用于覆盖掉 `material` 携带的物品描述 (Lore)。
+ 设为 `true`，则覆盖掉 material 的物品描述
+ 设为 `false`，则将会在 material 原有物品描述的基础上，在最后面添加图标配置的 lore 描述
```yaml
    reset-lore: false
```

## glow

(可选，默认为 `false`) 设置图标是否发光，即是否添加 `耐久` 附魔并隐藏附魔，使其拥有附魔光泽。
```yaml
    glow: true
```

## custom-model-data

(可选，默认为空) 设置图标的自定义模型数值，用于修改图标的模型与贴图。
```yaml
    custom-model-data: 10000
```

## nbt-strings

(可选，默认为空) 设置图标的自定义 NBT 标签，使用字符串类型。
```yaml
    nbt-strings:
      标签名: '标签值'
```

## nbt-ints

(可选，默认为空) 设置图标的自定义 NBT 标签，使用整数类型。
```yaml
    nbt-ints:
      标签名: 1
```

## needs-update

(可选，默认为 `false`) 设置该图标是否需要随菜单更新任务自动更新名称、描述等。
```yaml
    needs-update: true
```

## priority

(可选，默认为 `0`) 设置该图标的显示优先级，数值越小越优先显示，在该图标不满足显示条件时再去尝试下一个图标。
```yaml
    priority: 10
```

## view

(可选，默认为空) 设置查看图标条件，在条件不满足时不添加该图标到菜单，建议与优先级搭配使用。  
有关条件配置 (`requirement`) 相关事宜，详见 [菜单条件](/docs/inventory/requirments) 文档。
```yaml
    # 图标查看条件配置
    view: {}
    # 一种简写示例
    view-requirement: page has-prev-page
    # 一种简写示例
    view-requirments:
      权限: perm your.permission.here
      等级: eval %player_level% >= 30
```

## 点击事件

(可选) 设置图标的点击需求以及点击执行操作。所有类型的点击事件的格式都是相同的，仅有键名不同
+ `left-click` - 左键点击
+ `right-click` - 右键点击
+ `shift-left-click` - Shift+左键点击
+ `shift-right-click` - Shift+右键点击
+ `drop-click` - Q键点击
+ `ctrl-drop-click` - Ctrl+Q键点击

以下按左键点击进行示例，以 `commands` 结尾的是点击[执行操作](/docs/inventory/actions)，多条操作可以用列表，单条操作也可以用字符串。  
有关条件配置 (`requirement`) 相关事宜，详见 [菜单条件](/docs/inventory/requirments) 文档。
```yaml
    left-click:
      requirement: eval %vault_eco_balance% >= 100
      #requirements:
      #  金钱: eval %vault_eco_balance% >= 100
      deny-commands: '[message]<yellow>你没有足够的金钱购买这个商品'
      commands:
        - '[console]money take %player_name% 100'
        - '[message]<green>你购买了价值<yellow> 100金币 <green>的商品'
        - '[close]'
```
此外，还支持以下变体格式，在键名基础上添加后缀即可使用，以便适配不同需求。
```yaml
    left-click-requirement: eval %vault_eco_balance% >= 100
    #left-click-requirements:
    #  金钱: eval %vault_eco_balance% >= 100
    left-click-deny-commands:
      - '[message]<yellow>你没有足够的金钱购买这个商品'
    left-click-commands:
      - '[console]money take %player_name% 100'
      - '[message]<green>你购买了价值<yellow> 100金币 <green>的商品'
      - '[close]'
```
例如，你只想左键点击打开菜单，不需要改变缩进，就能添加点击操作。
```yaml
    left-click-commands: '[open]example'
```
